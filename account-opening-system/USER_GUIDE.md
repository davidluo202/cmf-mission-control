# 客户开户申请系统 - 用户指南

## 系统简介

这是一个完整的客户开户申请系统，支持个人客户通过11个步骤完成开户申请流程。系统采用现代化的Web技术栈，确保数据安全和用户体验。

## 主要功能

### 1. 用户认证
- 使用Manus OAuth登录
- 支持邮箱验证码验证（功能已实现，需集成邮件服务）

### 2. 开户申请流程（11个Case）

#### Case 1-2: 账户选择
- 选择客户类型：个人（联名和机构暂不可用，灰显）
- 选择账户类型：现金账户（保证金和衍生品暂不可用，灰显）

#### Case 3: 个人基本信息
- 中文姓名
- 英文姓名
- 性别
- 出生日期
- 出生地
- 国籍

#### Case 4: 个人详细信息
- 身份证件类型
- 证件号码
- 证件签发地
- 证件有效期（支持长期有效选项）
- 婚姻状况
- 学历
- 联系方式（电邮、电话、住址）

#### Case 5: 职业信息
- 就业状况：自雇/受雇/学生/无业
- 根据就业状况填写相应的详细信息

#### Case 6: 就业详情
- 收入来源
- 年收入范围
- 净资产范围

#### Case 7: 财务与投资
- 投资目的（多选）
- 投资经验（各类产品的经验年限）
- 风险承受能力

#### Case 8: 银行账户
- 支持添加多个银行账户
- 银行名称
- 账户币种
- 账户号码
- 开户人姓名（自动从Case 3导入）

#### Case 9: 税务信息
- 税务居住地（自动从国籍导入）
- 税务识别号（自动从证件号导入）

#### Case 10: 文件上传
- 身份证件正面（必须）
- 身份证件反面
- 银行月结单
- 住址证明
- 支持的格式：JPG, PNG, PDF
- 单个文件最大5MB

#### Case 11: 人脸识别
- 使用设备摄像头进行人脸识别
- 支持重新拍摄

#### Case 12: 监管声明和签署
- PEP（政治公众人物）声明
- US Person（美国人士）声明
- 阅读并同意开户协议（CMF001）
- 确认接受电子交易条例（ETO）约束
- 确认接受反洗钱（AML）和合规监管要求约束
- 电子签名（必须与Case 3的英文姓名一致）

### 3. 信息预览
- 展示所有已填写的信息
- 支持返回任意Case进行修改
- 保存并生成申请编号
- 提交申请

## 数据持久性

系统确保所有数据的持久性：
- 每个Case保存后立即写入数据库
- 前后导航时数据不会丢失
- 所有保存操作都有验证机制

## 技术特点

### 前端
- React 19 + TypeScript
- TailwindCSS 4 + shadcn/ui组件库
- 响应式设计，支持移动端和桌面端
- 蓝色主题，现代简洁的界面

### 后端
- tRPC 11（类型安全的API）
- Express 4
- Drizzle ORM
- MySQL/TiDB数据库

### 安全性
- 所有API都有用户权限验证
- 文件上传到S3安全存储
- 数据保存后立即验证确保不丢失
- 支持电子签名和监管合规

## 使用流程

1. **登录系统**
   - 点击"开始使用"按钮
   - 使用Manus账号登录

2. **创建新申请**
   - 在申请列表页面点击"新开户申请"按钮
   - 系统自动创建一个新的申请记录

3. **填写表单**
   - 按照11个Case的顺序依次填写
   - 每个Case填写完成后点击"下一步"保存并继续
   - 可以随时点击"上一步"返回修改

4. **预览和提交**
   - 完成所有Case后进入预览页面
   - 仔细核对所有信息
   - 点击"保存并生成申请编号"按钮
   - 获得申请编号后点击"提交申请"

5. **查看申请状态**
   - 在申请列表页面查看所有申请
   - 查看申请状态：草稿/已提交
   - 点击"继续填寫"继续未完成的申请
   - 点击"查看详情"查看已提交的申请
