## 問題發現 ⚠️

**審批人員評定風險等級顯示為空（"-"）**

這說明審批人員選擇的風險等級（R4）沒有被正確轉換為中文描述。

## 原因分析

在ApprovalDetail.tsx第835-841行，審批人員風險等級使用了`riskMap`來轉換：

```typescript
const riskMap: Record<string, string> = {
  'low': '低风险',
  'medium': '中等风险',
  'high': '高风险'
};
return riskMap[approvedRiskProfile] || '-';
```

但`approvedRiskProfile`的值是`R4`，不是`low`/`medium`/`high`，所以無法匹配，返回了`'-'`。

## 修復方案

需要更新審批人員風險等級的顯示邏輯，使用`getRiskToleranceDescription`函數來獲取R1-R5的完整描述。
