## 🔑 重要系统规范（必须遵守）

### 版本号格式
`v1.0.YYMMDD.NNN` — v1.0 冻结到生产上线前；YYMMDD=六位日期；NNN=三位每日序号

### 询价编号格式
`RFQ-YYMMDD-SSS` — YYMMDD=六位日期；SSS=三位每日序号（每天从001开始）
⚠️ 不能用 `Date.now().toString(36)` 这种随机格式！

### 交易状态机
```
新建询价
  ├─ 有人工勾选无报价表券商 → pending_quote (待上游报价)
  │    → 发询价邮件给无报价券商
  │    → 手工填入反馈费率
  │    → 汇总最低价+加点 → pending_confirm
  └─ 全部自动报价 → pending_confirm (报价待客户确认)
       → 生成客户报价（邮件/微信模板）
       ├─ 客户确认 → client_confirmed → 通知中标券商 → traded（入台账）
       └─ 客户拒绝 → rejected
```

### 券商ID映射
- CICC=1, CITIC(里昂,隐藏)=2, HTI=3, OI=4, UBS HK=5, CITIC Capital=6
- BROKER_QUOTE_STORAGE: `{ 1:'CICC', 3:'HTI', 5:'UBS', 6:'CITIC_CAP' }`
- 存储key前缀: `broker_quotes_v2_`

### 报价key格式
`${tenor}_${strikePercent}${direction}` 例如 `1M_100C`, `3M_80C`
- TENOR_MAP: `{ 30:'1M', 60:'2M', 90:'3M', 180:'6M', 360:'12M' }`
- 费率存储为比例(0.0507)，展示为百分比(5.07%)

### 报价表存储
- 个股: type='stock'
- ETF/股指: type='etf'
- 5开头→.SH; 1/0/3开头→.SZ
